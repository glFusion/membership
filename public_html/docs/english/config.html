glFusion Membership Plugin Configuration

The Membership Plugin allows site users to become members and gain access to additional features. This requires the PayPal plugin to be installed and working correctly.

Main Membership Options

Current Member Group
Select the glFusion group to which members will automatically be added as current members. This can be used to limit access to content, create lists, etc. As soon as a membership expires, the member will be removed from this group. Use this group to limit access to only current members.
Grace Period
Enter the grace period for membership expirations. This is the number of days after a membership expires during which the member will still be able to access the additional features. Enter "0" to have the membership terminate immediately upon expiration.
Days after expiration when member will be dropped
This is the number of days after expiration when a member's membership record will be considered "dropped" and will no longer be shown in the membership list. Enter "-1" to disable this feature, or zero to have memberships dropped upon expiration.

There is no automatic facility for deleting membership records completely from the database. If a membership is created in error it must be deleted manually.
Round expiration up to end of month?
Select whether to round the expiration date to the end of the month. For example, if a member joins on July 15, selecting "Yes" here will have their membership expire on July 31 of next year.
Early Renewal
This is the number of days prior to a membership's expiration when it may be renewed. You probably want to allow members to renew at least a few days before their memberships expire.
Days before expiration to send notifications
This is the number of days prior to expiration when notifications will be sent to members. Only one notification will be sent per expiration; when the membership is renewed the flag is reset to that an email will be sent before the new expiration date. Enter "-1" to disable notifications completely.
Method of expiration notifications
Select how members should be notified that their membership is about to expire. The options are none, via email, via message at login, or both.
Membership Period Start
Select the starting month for memberships, or "Rolling" if memberships can be started at any time.
Disable Expired Accounts?
Select "Yes" if you wish to have members' site accounts disabled when their membership expires. This option would be used if you operate a private site that only permits current members to log in. The account status will be set to "Banned". Rember that the administrator must manually re-enable the account for members who wish to renew after expiration.
Require App?
Select whether members are required or able to fill out an application.
View App?
Select whether members can view their application.
Display glFusion Blocks
Select the glFusion blocks to be displayed on the main membership page.
Show on glFusion Menu
Select whether an option to this plugin's home page should be placed on glFusion's "Plugins" menu.
Log debugging messages?
Select "Yes" to enable verbose logging to the glFusion error log. Select "No" to turn off logging. Memberships, expirations and cancellations are always logged to the plugin's log.
Default Administrator's View
Select the default view presented to membership administrators.
Terms and Conditions Acceptance
Select whether applicants are able or required to accept terms and conditions as part of the application.
Terms and Conditions URL
Provide the URL to the Terms and Conditions page. This can be an internal or external page, and you can use "%site_url%" to represent the base URL of your site.
Use Membership Numbers?
Select how you will be assigning membership numbers to your members.
  1. Not Used - membership numbers are not created or displayed.
  2. Free-Form - You can enter a membership number in the user' record.
  3. Auto-Gen - Membership numbers will be created for new members according to the selected format (below). These numbers can be overridden by the administrator for each member.
Membership Number Format
Enter a format string (see "sprintf() documentation" for the creation of membership numbers. The default "%04d" will create a membership number equal to the user ID but you can add text or override this.
You may also create a function in lib-custom.php named "CUSTOM_createMemberNumber($uid)" which will receive the user's ID and return a string of your choice.

Profile List Options

Include current members?
Select "Yes" to show current members on the profile list by default. This option may be combined with other options to show selected member types only. Select "No" to omit current members. If "No" is selected for all profile list options, then all site members will be displayed.
Include members in arrears?
Select "Yes" to show members who are in arrears on the profile list by default. This option may be combined with other options to show selected member types only. Select "No" to omit members in arrears. If "No" is selected for all profile list options, then all site members will be displayed.
Include expired members?
Select "Yes" to show expired members on the profile list by default. This option may be combined with other options to show selected member types only. Select "No" to omit expired members. If "No" is selected for all profile list options, then all site members will be displayed.
Include expired members?
Select "Yes" to allow members to pay by check (or cash, or other manual method). This causes a payment icon to be displayed on the plan list that links to a remittance coupon.

Manual Payment Options

Enable manual payments?
Select "Yes" to allow members to pay by check (or cash, or other manual method). This causes a payment icon to be displayed on the plan list that links to a remittance coupon.
Remit To
Enter the name to whom membership checks should be payable. This will be shown on the remittance coupon as "Make checks payable to..."
Remittance Address
Enter the address to which payments should be sent. HTML such as <br /> can be used to format the address for the remittance coupon.

Integrations

Update mailing list segment
If the Mailchimp plugin is installed, the Membership plugin can call it to update the list segment associated with different member statuses. You can then segment your mailing list by Current, Former and Non-Members. The segments must be set up in Mailchimp in advance for this to work.
Segment for active members
Enter the Mailchimp list segment to use for active (current) members.
Segment for members in arrears
Enter the Mailchimp list segment to use for members in arrears (expired but within the grace period).
Segment for expired members
Enter the Mailchimp list segment to use for expired members who have not renewed within the grace period.
Segment for non-members
Enter the Mailchimp list segment to use for members with no membership record. Includes members dropped from the database.
Manage the user album quota?
If the MediaGallery plugin is installed and enabled, the Membership plugin can directly update the user album quotas based on changes in membership status.
User album quota for members
Enter a number of megabytes for the user album quota for current and in-arrears members. This should be a positive number greater than the non-member quota below.
User album quota for members
Enter a number of megabytes for the user album quota for expired and non-members. This should be a positive number and less than the quota for members. If not, the Mediagallery update is disabled.
Enable PayPal Payments?
Select whether to use PayPal for payments, and whether to allow Buy-Now buttons. The reason you may wish to disable Buy Now buttons is if you offer additional products along with the membership, such as decals, patches, name tags, etc. After the buyer clicks "Add to Cart" they will be directed to your product catalog.
Recirect URL After Purchase
(Optional) Enter a URL to redirect the buyer after purchasing a membership. This may be the home page, or a Paypal shop page offering membership-related items.
